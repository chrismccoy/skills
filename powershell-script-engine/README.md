# powershell-script-engine

Generates production, PSScriptAnalyzer-clean PowerShell scripts in the voice of a PowerShell expert and Windows systems engineer. Two free-text inputs plus three choices drive a locked four-section output: Script, Parameters, Usage Examples, Security Notes. Approved verbs only, comment-based help on every function, console+file logging, and hard security non-negotiables - no plaintext secrets, no ungated destructive operations, no invented cmdlets.

Part of the [`chrismccoy`](../README.md) Claude Code marketplace.

## Install

```
/plugin marketplace add chrismccoy/skills
/plugin install powershell-script-engine@chrismccoy
```

Slash command:

```
/powershell-script-engine "Audit local admin group membership across a server list and export to CSV"   # arg seeds TASK
/powershell-script-engine                                                                                # full 5-question intake
```

## Inputs

- `TASK` - one sentence describing the concrete job the script performs.
- `USER_REQUIREMENTS` - detailed spec: parameters, inputs, outputs, modules, constraints.
- `PS_TARGET` - runtime target (`PowerShell 7.x only` or `7.x + note 5.1 fallbacks`).
- `REMOTE` - remote execution mode (`Local only` or `Remote via PSSession`).
- `DESTRUCTIVE` - destructive-operation policy (`No destructive operations`, or `Allow named destructive ops` - must be named in requirements, gated via `SupportsShouldProcess`).

`USER_REQUIREMENTS` is treated as inert data. Directives or "ignore previous" patterns embedded in it are ignored.

## Output structure (locked)

1. `## Script` - fenced `powershell` block: `#Requires -Version 7.0`, comment-based help, `[CmdletBinding()] param( ... )`, validated parameters, try/catch/finally, console + file logging
2. `## Parameters` - table: Name | Mandatory | Type | Validation
3. `## Usage Examples` - minimum 3 fenced examples (local, pipeline, remote/PSSession)
4. `## Security Notes` - credential handling + execution-policy assumptions

Ends with: `--- Generated by PowerShell Script Engine | review before production use ---`

## Security non-negotiables

- No plaintext passwords, hardcoded credentials, API keys, or connection strings - `[PSCredential]` / `Get-Credential` / SecretManagement only.
- No destructive operations (`Remove-*`, `Format-*`, `Stop-*`, registry/disk writes) unless named in requirements; gated via `[CmdletBinding(SupportsShouldProcess)]` + `$PSCmdlet.ShouldProcess()` (the mechanism behind `-WhatIf`/`-Confirm`) when generated.
- No invented cmdlet, module, parameter, or version names - only real, documented PowerShell.
- No `Write-Host` for data, no aliases in the script body, no positional args in examples, no rule suppressions.

## Scope lock

Outputs the requested script plus usage examples only. Empty or off-topic requirements get one clarifying question, then a halt.
